# K-Beauty Buyers Guide - 개발 계획서

> CCG 3모델 종합 검토 결과 기반 (Claude + Codex + Claude UX 분석)
> 작성일: 2026-03-24

---

## 1. 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 프론트엔드 | Next.js + TypeScript + Tailwind CSS | 반응형, SSR/SEO, 빠른 개발 |
| 백엔드/API | Next.js API Routes | 별도 서버 불필요, 단순화 |
| DB/인증/저장소 | Supabase (PostgreSQL + Auth + Storage) | 관리형, 비개발자도 운영 가능 |
| 배치/크롤링 | Cloud Run Jobs + Cloud Scheduler | 월간 자동 수집 |
| 검색/추천 | PostgreSQL + pgvector | 벡터 검색, 초기 비용 절감 |
| i18n (다국어) | next-intl + URL 기반 (/en, /ja, /zh...) | 7개국 SEO 최적화 |
| AI 추천 | 규칙기반 필터 + LLM 재랭킹/설명 | 정확성 + 설명 가능성 |
| AI 번역 | Claude/GPT API | 7개국 자동 번역 |
| 디자인 | AI 디자인 도구 + Tailwind UI 템플릿 | 디자이너 불필요 |
| 모니터링 | Sentry + Cloud Logging | 장애 감지 |
| 호스팅 | Vercel (프론트) + Supabase (백엔드) | 자동 배포, 무료 티어 |

## 2. 데이터베이스 핵심 테이블

### 사용자
- `users` — 기본 사용자 정보
- `user_profiles` — 국가, 언어, 관심 시술
- `social_accounts` — 소셜 로그인 연동

### 병원
- `clinics` — 병원 기본 정보
- `doctors` — 의사/원장 정보
- `procedures` — 시술 종류
- `clinic_procedure_prices` — 병원별 시술 가격

### 심평원 데이터
- `hira_raw_imports` — 원본 데이터 (버전 보관)
- `hira_metrics_monthly` — 월별 지표
- `clinic_hira_mapping` — 병원 매핑

### 크롤링
- `crawl_sources` — 크롤링 대상 사이트
- `crawl_jobs` — 크롤링 작업 기록
- `crawl_results` — 크롤링 결과

### AI 추천
- `recommendation_sessions` — 추천 요청
- `recommendation_candidates` — 후보 병원
- `recommendation_results` — 최종 추천 결과

### 광고
- `ads_campaigns` — 광고 캠페인
- `ads_impressions` — 노출 기록
- `ads_clicks` — 클릭 기록

### 커뮤니티
- `boards` — 게시판 카테고리
- `posts` — 게시글
- `comments` — 댓글
- `votes` — 추천/비추천
- `reports` — 신고

### 콘텐츠
- `before_after_assets` — 전후 사진
- `consent_records` — 초상권/동의 추적
- `translations` — 번역 데이터

### 관리
- `admin_users` — 관리자
- `audit_logs` — 감사 로그

## 3. AI 추천 엔진 설계

```
사용자 입력 (퀴즈)
    ↓
[1단계] 하드 필터 (SQL)
    - 예산, 지역, 시술 종류, 제재 여부
    ↓
[2단계] 점수화
    - 심평원 지표 + 가격 + 최신성 + 사용자 선호
    ↓
[3단계] 의미 매칭
    - 임베딩 유사도로 Top N 정제
    ↓
[4단계] LLM 재랭킹
    - Top 3 선정 + 추천 이유 생성
    ↓
[5단계] 광고 삽입
    - 추천 3개 + 광고 1개 (분리된 정책)
    ↓
결과 출력
```

## 4. 심평원 데이터 수집 파이프라인

```
Cloud Scheduler (월 1회 트리거)
    ↓
API 수집 (증분/체크포인트)
    ↓
크롤링 보강 (Puppeteer)
    ↓
Raw 저장 (JSON, 버전 보관)
    ↓
정규화 (병원명/주소/사업자번호 매핑)
    ↓
품질 검증 (누락/이상치/급변 탐지)
    ↓
PostgreSQL 업서트 (이력 보존)
    ↓
결과 리포트 자동 생성
    ↓
실패 시 재시도 + 알림 (이메일)
```

## 5. 개발 순서 (Phase별)

### Phase 0: 기획 확정 (1주)
- [ ] 추천 로직 KPI 정의
- [ ] 법적 면책 문구 확정
- [ ] 국가/언어 범위 확정
- [ ] 데이터 스키마 설계
- [ ] 크롤링 법적 검토 (robots.txt, 약관)

### Phase 1: 플랫폼 기반 (3주)
- [ ] Next.js + Supabase 프로젝트 셋업
- [ ] 회원가입 (소셜 로그인 + 이메일)
- [ ] 권한 시스템 (사용자/관리자)
- [ ] i18n 라우팅 (/en, /ja, /zh...)
- [ ] 기본 관리자 페이지
- [ ] 반응형 레이아웃 (Tailwind)

### Phase 2: 데이터 파이프라인 (3주)
- [ ] 심평원 API 수집기 구현
- [ ] 웹 크롤러 구현 (Puppeteer)
- [ ] 데이터 정규화/매핑 로직
- [ ] 월간 자동 배치 (Cloud Scheduler)
- [ ] 품질 검증 + 리포트
- [ ] 병원 DB 구축

### Phase 3: AI 추천 MVP (3주)
- [ ] 퀴즈/자가진단 UI
- [ ] 필터링 + 점수화 로직
- [ ] LLM 연동 (추천 이유 생성)
- [ ] Top 3 + 광고 1개 결과 페이지
- [ ] 추천 문의 기능
- [ ] 가격 비교표

### Phase 4: 커뮤니티 (3주)
- [ ] 레딧형 게시판 (카테고리별)
- [ ] 추천/비추천 (upvote/downvote)
- [ ] 댓글 시스템
- [ ] 전후 갤러리 (로그인 전용)
- [ ] 스팸/광고 방지
- [ ] 신고 기능

### Phase 5: 콘텐츠 & 성장 (3주)
- [ ] 의사 영상 자동 큐레이션 (유튜브 조회수 기준)
- [ ] 인플루언서 큐레이션 코너
- [ ] 트렌드 리포트
- [ ] 인기 시술 랭킹
- [ ] 비자/입국 안내
- [ ] FAQ 시스템
- [ ] 알림 시스템

### Phase 6: SEO & SNS (2주)
- [ ] 7개국 검색엔진 최적화
- [ ] hreflang 태그
- [ ] 구조화 데이터 (JSON-LD)
- [ ] 국가별 시술 키워드 랜딩 페이지
- [ ] SNS 공유 + 해시태그 캠페인
- [ ] 각국 SNS 연동

### Phase 7: 수익화 (2주)
- [ ] AI 추천 광고 (추천 3 + 광고 1)
- [ ] 배너 광고 시스템
- [ ] 카테고리 스폰서
- [ ] 프리미엄 추천 컨설팅
- [ ] 광고 성과 대시보드
- [ ] 광고비 청구/결제

## 6. 사용자 여정 (User Journey)

```
[첫 방문] 검색엔진/SNS로 유입
    ↓
[탐색] 인기 시술 랭킹, 트렌드 리포트 확인
    ↓
[관심] 퀴즈/자가진단 "나에게 맞는 시술은?"
    ↓
[추천] AI가 TOP 3 병원 추천 + 추천 이유
    ↓
[비교] 가격 비교표, 전후 갤러리 (로그인 유도)
    ↓
[회원가입] 소셜/이메일 간편 가입
    ↓
[심화] 커뮤니티 후기 확인, Q&A
    ↓
[문의] 추천 문의 또는 병원 직접 연락
    ↓
[재방문] 알림, 트렌드 업데이트로 재유입
```

## 7. 리스크 & 대응

| 리스크 | 심각도 | 대응 방안 |
|--------|--------|----------|
| 1인 개발 과부하 | 높음 | MVP 최소화, Phase별 순차 개발 |
| 크롤링 법적 이슈 | 높음 | robots.txt 준수, 공공 API 우선 |
| 의료 번역 오역 | 중간 | 의료 용어 사전 구축, 주요 문구 수동 검수 |
| LLM 비용 급증 | 중간 | 캐싱, 호출 제한, 규칙기반 우선 |
| 전후사진 초상권 | 높음 | 동의서 시스템, consent_records 테이블 |
| 병원 매핑 오류 | 중간 | 사업자번호 기준 매핑, 수동 보정 도구 |
| 허위 후기/스팸 | 중간 | 자동 필터 + 신고 + 게시 제한 |
| 중국 접속 차단 | 낮음 | 현재 대응 없이 진행, 향후 검토 |
